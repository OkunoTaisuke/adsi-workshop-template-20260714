# 勤怠管理システム — 要求仕様

## 概要

中小企業（〜50名規模）向けの Web ベース勤怠管理システム。
社員が日々の出退勤・休憩を打刻し、管理者が全社員の勤怠を確認・集計できる。

## ユーザーと権限

| ロール | できること |
|--------|-----------|
| 一般社員 | 打刻、自分の勤怠閲覧、打刻修正、有給・休暇申請 |
| 管理者 | 上記すべて＋全社員の勤怠確認、月次レポート出力、休暇申請の承認/却下 |

- 認証: メールアドレス + パスワード
- 識別子: メールアドレス

## 機能一覧

### F-1: 出勤・退勤の打刻

- Web 画面のボタン押下で打刻する
- 出勤・退勤の時刻を記録する
- 同日に二重出勤打刻はできない（退勤前に再度出勤は不可）

### F-2: 休憩開始・終了の打刻

- 休憩開始・休憩終了をボタンで記録する
- 1日に複数回の休憩を記録できる

### F-3: 打刻修正

- 本人が自分の打刻を直接修正できる（管理者承認は不要）
- 修正履歴を保持する（いつ・何から何に変更したか）

### F-4: 勤怠一覧の閲覧（自分の履歴）

- 日別の出勤・退勤・休憩・勤務時間を一覧表示する
- 月単位で表示を切り替えられる

### F-5: 管理者による全社員の勤怠確認

- 管理者が全社員の勤怠一覧を閲覧できる
- 社員名・日付で絞り込みができる

### F-6: 月次集計・レポート出力

- 月単位で勤務時間を集計する
- CSV 形式でダウンロードできる
- 集計項目: 社員名、勤務日数、総勤務時間、総休憩時間、残業時間

### F-7: 有給・休暇申請

- 社員が休暇種別・日付を指定して申請する
- 管理者が承認または却下する
- 申請ステータス: 申請中 → 承認 / 却下

## 勤務ルール

- 勤務形態: フレックスタイム制
- コアタイム: 10:00〜15:00
- フレキシブルタイム: コアタイム以外
- 所定労働時間: 8時間/日（コアタイム＋フレキシブルタイムの合計）
- 残業: 所定労働時間を超えた分

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| Backend | Java / Spring Boot |
| Frontend | TypeScript / Next.js |
| DB | PostgreSQL（テスト時: H2） |
| デプロイ | AWS (ECS + RDS) via CDK |

## スコープ外（初期リリースに含まない）

- シフト管理
- PDF レポート出力
- SSO / 外部認証連携
- GPS / ICカード打刻
- 部署管理機能

## 非機能要件

- 対応規模: 〜50人
- レスポンス: 画面操作は 2秒以内
- セキュリティ: パスワードはハッシュ化保存、HTTPS 通信
