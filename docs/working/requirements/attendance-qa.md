# 勤怠管理システム — 要求仕様 Q&A

## 基本スコープ

[Question] 対象ユーザーは誰ですか？（例: 中小企業の社員＋管理者、特定部署のみ、等）
[Answer]中小企業の社員＋管理者（全社対象。「特定部署のみ」は誤り）

[Question] 主な機能スコープはどこまでですか？以下から選択または追加してください:
  - 出勤・退勤の打刻
  - 休憩開始・終了の打刻
  - 勤怠一覧の閲覧（自分の履歴）
  - 管理者による全社員の勤怠確認
  - 月次集計・レポート出力（CSV等）
  - 有給・休暇申請
  - シフト管理
[Answer]シフト管理は不要

## 社員・認証

[Question] 社員の識別方法は？（自動採番ID / 社員番号手入力 / メールアドレス）
[Answer]メールアドレス

[Question] 認証方式は？（ID+パスワード / SSO / 認証なし（社内ネットワーク前提））
[Answer]ID+パスワード

[Question] ロール（権限）は何種類必要ですか？（例: 一般社員 / 管理者 の2種）
[Answer]一般社員 / 管理者

## 打刻

[Question] 打刻の方法は？（Web画面でボタン押下 / ICカード / GPS / その他）
[Answer]Web画面でボタン押下

[Question] 打刻の修正は可能ですか？（本人が修正申請 → 管理者承認 / 管理者が直接修正 / 修正不可）
[Answer]本人が修正可能

[Question] 1日の勤務パターンは？（固定時間制 / フレックス / シフト制 / 複数対応）
[Answer]フレックス

## 技術・制約

[Question] 技術スタックは以下で問題ありませんか？
  - Backend: Java / Spring Boot
  - Frontend: TypeScript / Next.js
  - DB: PostgreSQL（H2 for test）
  - デプロイ: AWS (ECS + RDS) via CDK
[Answer]問題ない

[Question] 初期リリースで対応する社員数の規模感は？（〜50人 / 〜500人 / 1000人以上）
[Answer]〜50人

## 追加確認

[Question] 「特定部署のみ」とのことですが、部署の管理機能（部署の追加・編集）は必要ですか？
  それとも初期データとして固定で登録しておけば十分ですか？
[Answer]特定部署のみは間違い

[Question] フレックスの場合、コアタイム（必須勤務時間帯）はありますか？
  例: 10:00〜15:00 はコアタイム、それ以外はフレキシブルタイム
[Answer]10:00〜15:00 はコアタイム、それ以外はフレキシブルタイム

[Question] 打刻修正を「本人が修正可能」とのことですが、管理者の承認なしで直接修正できる形でよいですか？
  （承認フローは不要？）
[Answer]承認無し

[Question] 有給・休暇申請のフローは？
  a) 本人が申請 → 管理者が承認/却下
  b) 本人が登録するだけ（承認不要）
[Answer]本人が申請 → 管理者が承認/却下

[Question] 月次レポートの出力形式は CSV のみで十分ですか？（PDF等は不要？）
[Answer]csvのみ
